## Light Terminal Background Compatibility

Text and overlay elements are now readable on light terminal backgrounds (e.g., Solarized Light, macOS Terminal white). Previously, card titles, directory labels, overlay text, and prompt elements used hardcoded `White`/`Gray`/`DarkGray` colors that were invisible or extremely low-contrast on light backgrounds.

The dashboard now routes all neutral text through a theme-aware `ColorPalette` that selects appropriate foreground colors for light and dark backgrounds. Terminal background is detected automatically via OSC-11 at startup, with a fallback to dark theme for terminals that do not support the query. An optional `theme` config key lets users override detection explicitly. Status indicator colors (Idle, Working, Error, Thinking) remain as ANSI accent colors that adapt via the terminal's own theme.

Readability is pinned by two new L1 snapshot tests (`theme/contrast/001` dark, `theme/contrast/002` light) covering all six overlay and prompt surfaces — stats bar, quit/stop confirmations, star prompt, config-gen prompt, and the No Agent card.
