## Terminal Bell Notifications

Get notified when agent sessions need attention without constantly watching the dashboard. The terminal bell fires when sessions transition to actionable states, letting your terminal emulator bounce the dock icon, flash the taskbar, or play a sound.

Bell notifications trigger on state transitions to WaitingForInput (agent needs a permission prompt answered) and Error (something went wrong). Idle transitions are off by default to avoid noise from frequent completions. Multiple simultaneous transitions coalesce into a single bell — no rapid-fire alerts. The bell only fires on actual state changes, never repeatedly for the same state.

Configure via `config.toml` under `[bell]`: toggle globally with `enabled`, or fine-tune per state with `on_waiting_for_input`, `on_idle`, and `on_error`. All options default to sensible values — bell works out of the box with no configuration needed.
