## Native Windows Support — Foundation

dot-agent-deck now builds and runs natively on Windows (`x86_64-pc-windows-msvc`) without requiring WSL. Windows users who previously had to configure a WSL layer to use the tool can now work directly in a native Windows environment.

The internal IPC layer has been rewritten behind a cross-platform abstraction (`src/platform/`). On Unix the behavior is identical to before — Unix domain sockets, `flock`, `getsockopt(SO_PEERCRED)`, and `setsid` are preserved unchanged. On Windows, named pipes replace Unix domain sockets (`\\.\pipe\dot-agent-deck-{user}-hook` / `-attach`), eliminating the entire class of stale-socket bugs; a Win32 named mutex replaces `flock` for spawn serialization; and `GetNamedPipeServerProcessId` / `GetNamedPipeClientProcessId` give zero-protocol peer-PID resolution for daemon stop and version handshake. The `libc` crate is now gated to `cfg(unix)` dependencies; `windows-sys 0.59` and `dirs 6` are added as `cfg(windows)` counterparts. A continuous `windows-latest` CI job (build + clippy + nextest) ensures Windows branches stay green on every commit.

**Note:** The Windows daemon is intentionally hard-failed (`Unsupported`) at this stage — a secure named-pipe security descriptor ships in the follow-up [#163](https://github.com/vfarcic/dot-agent-deck/issues/163). Full interactive e2e validation and release binaries (`.exe`, Scoop) ship in [#164](https://github.com/vfarcic/dot-agent-deck/issues/164).
