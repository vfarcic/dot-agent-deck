## New-agent command defaults to the last executed command

The new-agent form's Command field now pre-fills from the last interactive command you ran when no `default_command` is configured, eliminating the need to retype the same command on every spawn.

The seed follows a strict fallback chain: if `default_command` is set in your config it wins unconditionally (no change for existing users); otherwise the field pre-fills from the most recent command you submitted interactively; if no prior command exists (fresh install or cleared state) the field remains blank, preserving the current behavior. The value is global, persisted across deck restarts, and only reflects commands you typed yourself — authoring-mode commands spawned automatically by the scheduler and issue-dispatch are excluded so they don't pollute the fallback. The field remains editable; a wrong pre-fill costs a single clear.

See the [Configuration reference](https://agent-deck.devopstoolkit.ai/docs/configuration) for details on `default_command` and the fallback behavior.
