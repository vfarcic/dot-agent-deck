## Daemon-side pane_write trace instrumentation

Adds `RUST_LOG=pane_write=trace`-gated trace events across the daemon's PTY write path, providing byte-level observability into the orchestrator spawn-time role-prompt submission flow. This is Phase 1 of PRD #128's investigation into the unsubmitted spawn-time role prompt (a regression not resolved by PRD #100).

Four trace events are emitted at `target="pane_write"`, all inside the per-agent writer mutex, on two paths: the `write_to_pane_internal` path (payload write, submit terminator, notice terminator) and the `STREAM_IN` daemon-protocol forwarding path. Every event carries `pane_id` and `agent_id` fields to enable cross-path diffs between a failing spawn-time send and a working delegate send.

No behavior change — tracing is gated behind `RUST_LOG` and has no effect at the default log level. A new `escape_bytes_for_log()` helper in `pane_input.rs` provides hex-escaped output of non-printable bytes in trace payloads.
